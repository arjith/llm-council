## 3. Role Configuration

### 3.1 Council Builder

**Purpose:** Create and manage council configurations.

**Features:**
| Feature | Description | Priority |
|---------|-------------|----------|
| Drag-and-drop agent builder | Visual agent composition | P0 |
| Role presets | Common configurations (research, coding, debate) | P0 |
| System prompt editor | Customize agent behavior | P0 |
| Model selection | Choose from available models | P0 |
| Voting method selector | Configure consensus mechanism | P1 |
| Weight configuration | Set voting weights | P1 |
| Test council | Try configuration before saving | P1 |
| Import/Export YAML | Share configurations | P2 |

### 3.2 Agent Roles

**Primary Roles:**
| Role | Purpose | Typical Model |
|------|---------|---------------|
| **Reasoning Lead** | Complex analysis, synthesis | GPT-5, o3 |
| **Fast Responder** | Quick initial responses | GPT-5-mini, GPT-4o-mini |
| **Alternative View** | Different perspective | Claude, Gemini |
| **Domain Expert** | Specialized knowledge | Fine-tuned models |

**Backup/Support Roles:**
| Role | Activation | Purpose |
|------|------------|---------|
| **Verifier** | Always | Validates final output |
| **Tie-Breaker** | On voting tie | Resolves deadlocks |
| **Hot Standby** | On primary failure | Maintains availability |
| **Devil's Advocate** | On low confidence | Challenges consensus |
| **Sub-Agent Coordinator** | On complex tasks | Delegates sub-tasks |

### 3.3 System Prompt Templates

**Base Template:**
```
You are participating in an LLM Council as a {role}.

Your responsibilities:
- Provide thoughtful, well-reasoned responses
- Express confidence levels honestly
- Consider alternative viewpoints
- {role_specific_instructions}

When reviewing other responses:
- Evaluate objectively without bias
- Focus on accuracy, completeness, and reasoning
- Provide constructive rankings

Current council context:
- Topic: {topic}
- Other participants: {participant_count} models
- Your assigned perspective: {perspective}
```

**Role-Specific Templates:**
- Research Analyst: Emphasize evidence, citations
- Devil's Advocate: Challenge assumptions, find weaknesses
- Domain Expert: Leverage specialized knowledge
- Synthesizer: Combine and reconcile viewpoints

### 5.1 Quality Metrics

| **Correction Rate** | How often correction needed | Corrections / total responses |
